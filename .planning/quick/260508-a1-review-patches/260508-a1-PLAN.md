## Task 1: New Test Files — Property Tests + Audit Integration Test

Two new test files. No existing file modifications.

### 1a. Create `tests/property/test_probabilities.py`

**What:** Hypothesis-based property tests for core probabilistic invariants.

Tests to write (targets existing schemas/classes):
- **DirectionalProbability sum**: For any valid (p_up, p_flat, p_down) with sum=1.0 +/- 1e-6, model accepts. For sum outside range, `ValueError`. For degenerate (all mass on one direction), `ValueError`.
  - Imports: `pmacs.schemas.agents.DirectionalProbability, PersonaName`
- **FX round-trip**: For any positive amount and valid rate, `usd_to_eur(eur_to_usd(amount, rate), rate) == amount` within 1e-6.
  - Imports: `pmacs.schemas.currency.FxRate, usd_to_eur, eur_to_usd`
- **Freshness never mutates**: For any EvidencePacket with any fetched_at, `check_freshness()` returns a `FreshnessResult` and the packet's `fetched_at` is unchanged afterward.
  - Imports: `pmacs.data.staleness.check_freshness`, `pmacs.schemas.freshness.CriticalityLevel`, `pmacs.schemas.data.EvidencePacket`
- **Conviction verdict thresholds**: For any score in [0,1], `score_to_verdict(score)` returns correct tier (STRONG_BUY >= 0.6, BUY >= 0.3, SKIP < 0.3).
  - Imports: `pmacs.schemas.conviction.ConvictionResult, VerdictTier`
- **Arbitrated probabilities**: For any valid PROCEED decision, p_up+p_flat+p_down == 1.0 +/- 1e-6. For ABORT decisions, no sum constraint enforced.
  - Imports: `pmacs.schemas.arbitration.Arbitrated, ArbitrationDecision`

**Dependencies:** `hypothesis` must be in dev deps (add to pyproject.toml if missing).

### 1b. Create `tests/integration/test_audit_writing.py`

**What:** Integration test verifying state machine transitions emit audit log entries.

Tests to write:
- **Transition writes audit**: Call `state_machine.transition()` with a real `AuditWriter` (tmp_path). Verify audit log file gains one line with `event_type="holding_state_transition"`, correct `cycle_id`, valid hash chain.
- **Audit structure correct**: Parse written line. Verify 5 tab-separated fields. Verify `canonical_json` payload contains `from_state`, `to_state`, `reason`, `holding_id`.
- **Hash chain valid after transition**: Run 3 transitions. Run `AuditVerifier.verify_full()`. Assert passes.
- **Idempotent replay does not duplicate**: Same transition call twice (second should raise `InvalidStateTransition` from terminal state guard or state constraint). Verify audit log has exactly the expected number of entries.

**Imports:** `pmacs.engines.state_machine.transition`, `pmacs.storage.audit.AuditWriter, AuditVerifier`, `pmacs.schemas.contracts.*`

## Task 2: Patch Existing Source Files — Keychain, Logger, FX, Currency

Four files to patch. Each patch is self-contained.

### 2a. Patch `pmacs/storage/keychain.py`

**Current state:** Basic `get_api_key()` / `set_api_key()` with `KeychainError`. No error recovery, no secret scrubber, no rotation.

**Patches to apply:**

1. Add `_scrub_secrets(message: str, secrets: list[str]) -> str`:
   - Replaces each secret substring with `***REDACTED***`
   - Used by all error paths before logging/raising

2. Add `rotate_api_key(service, account, old_key, new_key) -> None`:
   - Call `get_api_key()` to verify `old_key` matches current
   - Call `set_api_key()` with `new_key`
   - Raise `KeychainError` if `old_key` doesn't match
   - Scrub keys from any exception messages

3. Wrap existing functions with scrubbed error messages:
   - All `KeychainError` messages scrubbed so service names visible but never key values
   - Add docstring notes about error recovery behavior (startup degraded mode, runtime cycle abort) for callers to implement

4. Add error codes to module-level constants:
   - `KEYCHAIN_UNAVAILABLE = "KEYCHAIN_UNAVAILABLE"`
   - `KEYCHAIN_RUNTIME_FAILURE = "KEYCHAIN_RUNTIME_FAILURE"`
   - (These already exist in error_classifier.py, so just reference them)

### 2b. Patch `pmacs/logsys/debug_log.py`

**Current state:** `log_debug()` already enforces `error_code` for WARN+. No `cycle_id` enforcement.

**Patches to apply:**

1. Add `SYSTEM_EVENT_TYPES` frozenset:
   - `{"PROCESS_START", "PROCESS_SHUTDOWN", "BOOT_DETECTED", "CONFIG_LOADED", "KILL_SWITCH_STATE"}`
   - These are the only events where `cycle_id=None` is acceptable

2. Add `cycle_id` runtime enforcement:
   - If `cycle_id is None` AND `event_type` not in `SYSTEM_EVENT_TYPES`: raise `ValueError` with message citing Architecture.md §5.2
   - This makes `cycle_id` a runtime requirement for all cycle-scoped events

### 2c. Patch `pmacs/data/fx.py`

**Current state:** Has `fetch_ecb_rate()` and helper functions. No staleness handling for weekends/holidays.

**Patches to apply:**

1. Add `_is_ecb_business_day(d: date) -> bool`:
   - Returns False for Saturday/Sunday
   - Returns False for known ECB holidays (New Year, Good Friday, Easter Monday, Labour Day, Christmas, Boxing Day — hardcode the set, ~10 days)
   - Returns True otherwise

2. Add `_last_ecb_business_day(d: date) -> date`:
   - Walk backward from `d` until `_is_ecb_business_day()` is True
   - Returns that date

3. Add `is_rate_stale(rate: FxRate, now: date | None = None) -> bool`:
   - If `now` is None, use `date.today()`
   - Compute `expected_business_date = _last_ecb_business_day(now)`
   - If `rate.business_date < expected_business_date`: stale (rate is older than most recent publication)
   - If `rate.business_date == expected_business_date`: fresh
   - Weekend/holiday logic: Saturday/Sunday rates are "fresh" if they match Friday's business_date

4. Update `fetch_ecb_rate()` docstring to note staleness handling

### 2d. Patch `pmacs/schemas/currency.py`

**Current state:** `FxRate` already has `_no_eur_per_usd_field` model_validator that checks `model_fields`. Already rejects the field.

**Patches to apply:**

1. Strengthen the validator to also check `model_dump()` keys:
   - Current check only looks at `model_fields` (declared fields). A subclass or dynamic construction could bypass this.
   - Add `if "eur_per_usd" in self.model_dump()` as belt-and-suspenders

2. No other changes needed — validator already works. This is a hardening patch.

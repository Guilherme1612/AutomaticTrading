  <name>Task 1: Wire main_loop() to MutationDaemon.run_cycle()</name>
  <files>pmacs/mutation/daemon.py</files>
  <action>
Replace the skeleton `main_loop()` body (lines 435-469). The current code already instantiates MutationDaemon (line 451) but the while loop just sleeps (line 466-469). Fix:

1. After creating the daemon instance, enter the while loop.
2. On each iteration: read the current paper cycle count from SQLite `cycles` table (`SELECT COUNT(*) FROM cycles WHERE state = 'CLOSED'`). If the table doesn't exist or query fails, default to 0.
3. Generate a cycle_id using `f"mutation-{int(time.time())}"`.
4. Call `daemon.run_cycle(cycle_id, paper_cycle_count)`.
5. Sleep 60 seconds (existing behavior).
6. Wrap the loop body in try/except logging MUTATION_DAEMON_LOOP_ERROR via log_debug with error_code="MUTATION_DAEMON_LOOP_ERROR" -- this prevents a single iteration failure from killing the daemon.
7. Add a log_debug MUTATION_DAEMON_ITERATION at INFO level at the start of each iteration with cycle_id and paper_cycle_count.

The existing imports (time, log_debug, sqlite3, Path) are sufficient. Do NOT add new imports.
  </action>

Add outcome persistence:

1. In `record_outcome()`: after appending to the in-memory list, also INSERT into `mutation_outcomes` table: `(proposal_id, cycle_id, arm, metric_name, metric_value, recorded_at)`. The cycle_id is not currently available in record_outcome -- add an optional `cycle_id: str = ""` parameter. Use `datetime.utcnow().isoformat()` for recorded_at. Wrap in try/except to be resilient if table doesn't exist.

2. In `_restore_from_db()`: after recovering active tests from mutation_proposals (existing code at line 100-117), also load outcome data from mutation_outcomes table: `SELECT arm, metric_value FROM mutation_outcomes WHERE proposal_id = ?`. 3. Add a `_ensure_outcomes_table()` private method that runs `CREATE TABLE IF NOT EXISTS mutation_outcomes (proposal_id TEXT, cycle_id TEXT, arm TEXT, metric_name TEXT, metric_value REAL, recorded_at TEXT)`. Call it in `__init__` when db_path is not None, before `_restore_from_db`.

Do NOT change the existing _persist_start or _persist_complete methods -- they handle status correctly. Do NOT change the method signature for start(), complete(), get_state(), can_start() -- only add the optional cycle_id param to record_outcome.

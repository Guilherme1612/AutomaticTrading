# ADR-010: No Backtesting Against Historical LLM Outputs

## Status

Accepted

## Context

Backtesting is the industry-standard method for validating trading strategies: run the strategy against historical data and measure what would have happened. PMACS could, in principle, feed historical EOD data through the persona pipeline and compare the resulting conviction scores against actual price movements.

However, PMACS's persona outputs are generated by a large language model (Qwen3.6-35B-A3B) whose training data includes information up to its knowledge cutoff. For any historical period within the training window, the model has already "seen the future." A backtest against 2024 earnings reports would be testing whether the model can recall what happened, not whether it can predict it.

This is an epistemic problem, not a technical one. The backtest would show artificially good results (because the model knows the outcomes) and provide false confidence in the system's predictive ability.

## Decision

PMACS does not backtest against historical LLM outputs. This is listed as a forbidden anti-pattern in Architecture.md §16.

The only valid forward-test is SHADOW mode: run the full pipeline on live data without execution, then compare predictions against actual outcomes as they unfold. SHADOW has been running since day 1 (see ADR-007), accumulating real forward-test data.

Source maturation and Brier scores are computed from live PAPER/SHADOW cycles only, not from retroactive analysis.

## Consequences

**Positive:**

- Epistemic honesty: the system's performance metrics reflect actual forward predictive ability, not retroactive recall.
- No false confidence from inflated backtest results. The operator makes promotion decisions (PAPER to PAPER_VALIDATED, etc.) based on real forward performance.
- Simpler system: no backtesting infrastructure, no historical data replay pipeline, no comparison engine.

**Negative:**

- The system needs ~90 PAPER cycles before promotion to PAPER_VALIDATED is possible. Without backtesting, this calibration period cannot be accelerated. At 1 cycle/day, that is ~3 months of operation before real money.
- No way to evaluate "what would have happened if we had used a different prompt/weight/threshold" on historical data. The Mutation Engine's A/B tests run forward-only in SHADOW mode.
- Industry convention expects backtesting. The operator must internally justify the absence when evaluating system confidence.
